# ফিবোনাচ্চি (ডাইনামিক প্রোগ্রামিং)

**ডাইনামিক প্রোগ্রামিং (Dynamic Programming)** বা **DP** হলো এমন একটি কম্পিউটার প্রোগ্রামিং কৌশল যা কোনো জটিল বা বড় সমস্যাকে ছোট ছোট উপ-সমস্যা বা সাব-প্রবলেমে (Sub-problems) বিভক্ত করে সমাধান করতে সাহায্য করে। এই কৌশলে প্রতিটি সাব-প্রবলেম মাত্র একবারই সমাধান করা হয় এবং তার ফলাফল একটি টেবিলে (যেমন- Array বা Hash Map) সংরক্ষণ বা মেমোইজ (Memoize) করে রাখা হয়, যাতে ভবিষ্যতে ওই একই হিসাব পুনরায় করতে না হয়।

ফিবোনাচ্চি ধারাটি ডাইনামিক প্রোগ্রামিংয়ের মূল নীতি বোঝার জন্য সবচেয়ে উপযুক্ত একটি উদাহরণ।

---

## সাধারণ রিকার্সন বনাম ডাইনামিক প্রোগ্রামিং (Recursion vs DP)
সাধারণ রিকার্সিভ পদ্ধতিতে ফিবোনাচ্চি সংখ্যা বের করতে গেলে একই ফাংশন বারবার কল করতে হয় (Overlap)। 
যেমন, $F(5)$ বের করার জন্য আমাদের প্রয়োজন:
* $F(5) = F(4) + F(3)$
* $F(4) = F(3) + F(2)$
* $F(3) = F(2) + F(1)$

এখানে লক্ষ্য করুন, $F(3)$ এর মান বের করার জন্য দুইবার এবং $F(2)$ এর মান তিনবার আলাদাভাবে রিকার্সিভ কল করে হিসেব করা হচ্ছে। এই অতিরিক্ত গণনার দরুন টাইম কমপ্লেক্সিটি দাঁড়ায় $O(2^n)$ যা অনেক বেশি।

কিন্তু ডাইনামিক প্রোগ্রামিং (ট্যাবুলার বা মেমোইজেশন) এর মাধ্যমে আমরা নিচের মতো করে এগোই:
* আমরা একটি টেবিল বানাবো: \`dp = [০, ১, null, null, null, null]\`
* ক্রমান্বয়ে বাম দিক থেকে ডানে হিসেব করে মানগুলো বসাবো।
* $F(2) = F(1) + F(0) = 1 + 0 = 1$ (সংরক্ষণ করি)
* $F(3) = F(2) + F(1) = 1 + 1 = 2$ (পূর্বের সংরক্ষিত মান ব্যবহার করি)
* এভাবে প্রতি ধাপের হিসাব মাত্র একবারই হবে! এর ফলে টাইম কমপ্লেক্সিটি নেমে আসে মাত্র $O(n)$-এ।

## বাস্তব জীবনের উদাহরণ (Real-world Example)
ধরুন আপনি একটি সিঁড়ি বেয়ে ৫ তলায় উঠবেন। প্রতি পদক্ষেপে আপনি ১টি বা ২টি ধাপ উপরে উঠতে পারেন। আপনি কতটি উপায়ে ৫ তলায় উঠতে পারবেন?
* এটি আসলে একটি ফিবোনাচ্চি ধারা মেনে চলে। 
* আপনি যদি প্রতিটি সিঁড়ির ধাপে পৌঁছানোর সমস্ত উপায় আগে থেকেই একটি কাগজে লিখে রাখেন (Memoization), তাহলে নতুন ধাপে পৌঁছানোর হিসেব করতে আপনাকে পুরোনো ধাপের হিসাবগুলো পুনরায় গুনতে হবে না। স্রেফ পেছনের দুটি হিসাব যোগ করলেই হবে।

## এটি কীভাবে কাজ করে? (How it works?)
ট্যাবুলার বা বটম-আপ (Bottom-Up) পদ্ধতিতে ধরি $N = 6$:

**ধাপ ১ (বেইজ কেস):**
* $dp[0] = 0$
* $dp[1] = 1$
* টেবিল: \`[০, ১, null, null, null, null, null]\`

**ধাপ ২:**
* $dp[2] = dp[1] + dp[0] = 1 + 0 = 1$
* টেবিল: \`[০, ১, ১, null, null, null, null]\`

**ধাপ ৩:**
* $dp[3] = dp[2] + dp[1] = 1 + 1 = 2$
* টেবিল: \`[০, ১, ১, ২, null, null, null]\`

**ধাপ ৪:**
* $dp[4] = dp[3] + dp[2] = 2 + 1 = 3$
* টেবিল: \`[০, ১, ১, ২, ৩, null, null]\`

**ধাপ ৫:**
* $dp[5] = dp[4] + dp[3] = 3 + 2 = 5$
* টেবিল: \`[০, ১, ১, ২, ৩, ৫, null]\`

**ধাপ ৬:**
* $dp[6] = dp[5] + dp[4] = 5 + 3 = 8$
* টেবিল: \`[০, ১, ১, ২, ৩, ৫, ৮]\`

ফলাফল পেয়ে গেছি! $F(6) = 8$।

## সুবিধা (Advantages)
* সময়ের বিশাল সাশ্রয় ঘটে: টাইম কমপ্লেক্সিটি $O(2^n)$ থেকে কমে $O(n)$ হয়ে যায়।
* সহজে বাস্তবায়ন করা যায় (বিশেষ করে ইটারেটিভ বা বটম-আপ উপায়ে)।
* এটি একই সাব-প্রবলেম বারবার হিসেব করা বন্ধ করে কার্যক্ষমতা বৃদ্ধি করে।

---

## অসুবিধা (Disadvantages)
* এতে মেমরি স্পেস বেশি লাগে (Space Complexity: $O(n)$) কারণ টেবিলটি স্টোর করতে হয়। অবশ্য এটিকে অপ্টিমাইজ করে কেবল শেষ ২টি উপাদান মনে রেখে স্পেস ওর্স্ট-কেস $O(1)$-এ নামিয়ে আনা সম্ভব।
